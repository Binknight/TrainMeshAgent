基于最新的openai agent sdk设计一个AI agent，该agent主要供AI训练组网仿真测试人员测试，以及验证模型等效使用，它支持一下特性：
1. agent 以web服务的形式对外提供服务，测试人员通过web页面类似cursor ide 的agent交互界面，以自然语言交互形式完成AI组网仿真及测试验证任务
2. 测试人员输入原始组网、以及等效后组网的关键信息，比如设备类型（A2/A3/A5）、DP（数据并行）、TP(张量并行)、PP(流水线并行)，agent调用组网创建的相关skill，可以快速生成json形式的组网图
    2.1 agent必须添加输入护栏，确保提供的输入比如设类类型、DP、TP、PP符合规范
    2.2 实现 training-mesh-gen-skill， 该skill负责根据输入，可以输出结构化的json组网对象
    2.3 agent必须添加输出护栏，检查输出的json组网是否满足要求, 如不满足，需要再次生成
3. 等效组网输出后，前端分别渲染原始组网和等效后的组网，测试人员检查完毕，确认可以继续仿真任务，agent通过仿真系统验证组网的单卡计算强度、内存占用、通信流量是否等效
    3.1 agent基于当前会话的原始组网、等效后组网的建模，以及默认的其他训练脚本参数，通过MCP client向仿真系统下发任务
    3.2 仿真系统是一套独立的可运行的python程序，通过MCP server暴露相关的可供agent调用的工具接口，比如仿真任务执行、状态上报、日志同步、结果查看、单卡详情等
4. agent 支持基于相关skill对大小组网的仿真结果，进行智能对比分析
    4.1 实现组网性能分析skill，名称training-mesh-profiler-skill，支持通过调MCP cil获取仿真结果及单卡详情，通过计算公式得出计算强度、内存占用、通信流量等指标，结构化输出
    4.2 agent分别针对大小组网（原始组网、等效组网）各调用一次该skill，得出两者的仿真结果，对比分析，验证等效模型是否有效（误差范围界定）、以及与理论计算值的对比
